Architecting Reusable Charts

Extensible

Using d3.chart

// define a chart
d3.chart("CircleChart", {
  initialize: function() {
    log("Hi! I'm a chart");
  }
});

// extend the original chart with additional functionality
d3.chart("CircleChart").extend("OtherCircleChart", {
  initialize: function() {
    log("I am a circle chart based on the one above!");
  }
})

var chart = d3.select(output)
  .chart("OtherCircleChart");
source: http://misoproject.com/d3-chart/api.html#d3chart_extend

Architecting Reusable Charts

Repeatable

Easily reuse a chart's blueprints on any node (using d3.chart)

// define a chart
d3.chart("CircleChart", {
  initialize: function() {
    log("Hi! I'm a chart");
  }
});

var chart1 = d3.select("#chart1")
  .chart("CircleChart");

var chart2 = d3.select("#chart2")
  .chart("CircleChart");
  • almost all of our implementations need to be easily updated with new data points
  • use d3's concept of stages
  • that's more boilerplate in all of our charts
  • d3.chart provides a 'draw' method that allows you to pass in a dataset those blueprints are expected to draw
  • your blueprints can have instructions for transitioning those stages (enter/exit/udpate)
  • blueprint also emits events you can use outside of your chart

Architecting Reusable Charts

Updatable

// assume previously defined d3.chart called
// CircleChart

// create an instance of the chart on a d3 selection
var chart = d3.select(output)
  .append("svg")
  .attr("height", 30)
  .attr("width", 400)
  .chart("CircleChart");

var data = [1,4,6,9,12,13,30];

// render it with some data
chart.draw(data);

// add data
data.push(35);

// render it with updated data
chart.draw(data);
source: Miso Project – d3.chart http://misoproject.com/d3-chart/api.html#d3chart_i_draw
